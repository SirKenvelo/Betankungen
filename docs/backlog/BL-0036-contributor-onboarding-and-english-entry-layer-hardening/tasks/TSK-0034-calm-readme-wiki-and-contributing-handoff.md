**Stand:** 2026-04-14

# Task
Die Rollen von README, GitHub Wiki und `CONTRIBUTING.md` so aufeinander
abstimmen, dass Erstbesucher und gelegentliche Contributors ohne zirkulaere
Pfadlogik oder doppelte Erwartungssignale in den jeweils richtigen
Einstiegsraum weitergeleitet werden.

# Notes
- Fokus ist der Handoff zwischen Einstiegsartefakten: wer fuehrt ein, wer
  erklaert oeffentlichen Schnellstart, wer uebernimmt Beitragspfad und Regeln.
- Der Slice soll Navigations- und Rollenfriktion senken, nicht neuen
  Contributor-Inhalt oder neue englische Substanz erfinden.
- Kompakte Contributor-Checkliste gehoert in `TSK-0032`.
- Englische Signalpflege und Sprachrahmung gehoeren in `TSK-0033`.
- Ziel ist ein ruhigerer Einstieg, nicht ein weiterer grosser Inhaltsumbau.

# Done When
- [x] README, Wiki und `CONTRIBUTING.md` haben klar erkennbare und
  nicht-widerspruechliche Rollen im Entry-Layer.
- [x] Zirkulaere oder doppelte Einstiegspfade fuer Erstbesucher sind reduziert.
- [x] Die Dokumente verweisen mit ruhiger Reihenfolge aufeinander, statt
  dieselbe Aufgabe mehrfach gleichzeitig zu beanspruchen.
- [x] Der Slice bleibt getrennt von neuem Contributor-Quickstart-Inhalt und
  von englischer Signalpflege.
