# P080
**Stand:** 2026-04-01

Stations-Masterdata-Policy-Block fuer offensichtliche Plausibilitaetsfehler in
`--add stations`/`--edit stations`.

Abgedeckte Regeln:
- `P-080`: offensichtliche Feldverschiebung zwischen `zip` und `city`
- `P-081`: `zip` muss numerisch sein (4..10 Ziffern)
- `P-082`: `city` darf nicht rein numerisch sein (Buchstaben erforderlich)
- `P-083`: `house_no` muss mindestens eine Ziffer enthalten
- `P-084`: `phone` darf bei gesetztem Wert nicht ziffernlos sein
- `P-085`: `latitude` und `longitude` muessen gemeinsam gesetzt oder gemeinsam leer sein
- `P-086`: `latitude` muss als Dezimalgrad im Bereich `[-90, 90]` vorliegen
- `P-087`: `longitude` muss als Dezimalgrad im Bereich `[-180, 180]` vorliegen
- `P-088`: `plus_code` akzeptiert entweder einen vollen Open Location Code oder
  einen lokalen/short Code mit gesetzter `latitude`/`longitude`; Eingaben
  werden vor der Persistenz normalisiert und als voller Code gespeichert

- Negative Faelle: Exitcode `!=0`, klare Policy-ID im Fehlertext, kein Insert.
- Positive Referenz: Exitcode `0`, Datensatz wird inkl. normalisierter
  Geodaten gespeichert.
